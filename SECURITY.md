## Security Policy (HDGP-Core)

This repository hosts the open-source **Meta** baseline materials (semantics, contracts, and governance). We still welcome and value **private** security vulnerability reports.

### Preferred reporting channel

- **GitHub Private vulnerability reporting** (preferred): use the private security reporting entry on this repository’s GitHub page.

### Alternative channels

- **Security email**: TBD  
- **PGP key / fingerprint**: TBD  

> Until a security email/PGP is published, please use GitHub private reporting. Do not disclose exploitable details in public Issues.

### Support scope

- **Supported versions**: the latest commit on the `main` branch is supported by default.  
- **Tags / releases**: if tags are introduced later, the latest tag will be added to the support scope; until a stable tagging policy exists, `main` is the reference.

### Response targets (SLA)

- **Note**: the following are target values; the disclosure channel is GitHub Private vulnerability reporting.  
- **Critical**: first acknowledgement ≤ 5 business days  
- **High**: first acknowledgement ≤ 10 business days  
- **Medium / Low**: first acknowledgement ≤ 20 business days

> These targets cover “acknowledgement / first response”. Remediation timelines will be communicated based on scope and governance processes.

## 安全政策（HDGP-Core）(ZH-CN)

本仓库用于公开的 Meta 伦理安全基线材料。我们仍然欢迎并重视安全问题的私密披露。

### 首选披露渠道

- **GitHub Private vulnerability reporting**（首选）：请使用本仓库 GitHub 页面中的安全漏洞私密报告入口提交。

### 备选渠道

- **安全邮箱**：TBD  
- **PGP 公钥/指纹**：TBD  

> 注意：在安全邮箱/PGP 尚未公布前，请优先使用 GitHub 私密报告渠道；不得向 Issue 公开披露可被立即利用的细节。

### 支持范围

- **支持版本**：默认支持 `main` 分支最新提交。  
- **Tag/Release**：如后续发布 tag，将补充“最新 tag”为支持范围；在 Core 尚未建立稳定 tag 策略前，以 `main` 为准。

### 响应目标（SLA）

- **渠道说明**：以下均为目标值；披露渠道以 GitHub Private vulnerability reporting 为准。  
- **Critical**：首次确认 ≤ 5 个工作日  
- **High**：首次确认 ≤ 10 个工作日  
- **Medium / Low**：首次确认 ≤ 20 个工作日

> 以上为“确认收到/首次响应”的目标值；后续修复/缓解时程将根据影响范围与治理流程另行沟通。
